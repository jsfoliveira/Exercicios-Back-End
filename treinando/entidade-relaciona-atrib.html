-- entidades, atributos e relacionamentos -- ENTIDADES são armazenadas em formato de tabelas em um banco de dados. Exemplo: album. -- ATIBUTO é tudo aquilo que pode ser usado para descrever algo. Exemplo: Artista: `artista_id` e `nome`. -- RELACIONAMENTO representa como uma entidade deve estar ligada com outra no banco de dados. Tem 3 tipos: -- Relacionamento Um para Um (1..1): uma tabela só tem 1 coluna igual a outra tabela -- Relacionamento Um para Muitos ou Muitos para Um (1..N): uma tabela tem mais de 1 coluna igual a outra tabela -- Relacionamento Muitos para Muitos (N..N): uma tabela (tabela auxiliar ou tabela junção) tem várias colunas iguais a várias tabelas
